# 账号实时指标显示设计

## 目标

在管理端凭据列表中补充至少五项可直接读取的运行指标，重点显示每个账号最近 1 分钟的 RPM，并让管理员能快速判断账号是否可用、是否接近限流、余额是否新鲜以及 Token 是否即将过期。

## 方案

采用“现有状态接口 + 前端派生指标”的方案。后端 `/api/admin/credentials` 已提供 `rpmCurrent`、`rpmLimit`、`inFlight`、成功/失败累计数、`expiresAt`、`balanceUpdatedAt`、`hasProxy` 等字段，本次不新增数据库字段、不修改客户请求链路、不改变调度或计费。

账号行在身份徽章下方增加紧凑指标带，桌面端横向排列，窄屏自动换行。每个指标带有标签、数值和 tooltip，颜色只作为辅助，文字同时表达状态。

新增展示指标：

1. **近 1 分钟 RPM**：直接显示最近 60 秒滚动窗口请求数；有限速时追加 `/ 上限`。
2. **RPM 使用率**：有限速时计算 `rpmCurrent / rpmLimit`，不限速显示“不限速”。
3. **成功率**：使用 `successCount / (successCount + totalFailureCount)`；没有请求时显示“暂无数据”。
4. **进行中**：显示当前 in-flight 请求数，便于发现卡住或并发堆积。
5. **Token 状态**：根据 `expiresAt` 显示有效、剩余时间或已过期。
6. **余额新鲜度**：根据 `balanceUpdatedAt` 显示多久前更新；没有缓存时显示“未查询”。
7. **连接方式**：显示“代理”或“直连”，避免管理员误以为账号都走同一路径。

## 刷新与性能

凭据查询从 30 秒改为 10 秒轮询，让 60 秒 RPM 和 in-flight 数保持可用的新鲜度。请求仍复用现有查询，不新增轮询接口；窗口不可见时遵循 React Query 的既有行为，不执行额外计算。

## 兼容与错误处理

- 所有新增字段均使用现有可选字段或安全默认值，旧服务响应不会导致页面崩溃。
- 无效时间戳、负数或除零情况显示明确的“未知/暂无数据”，不显示误导性的 0%。
- 指标纯展示，不改变启用/禁用、优先级、RPM 限制、Token 刷新、代理选择和余额查询行为。
- 不展示敏感 Token、完整邮箱或完整代理 URL。

## 验证

- 新增指标格式化函数的单元测试，覆盖不限速、零请求、零分母、过期 Token、余额未查询和时间边界。
- 新增 UI 合同测试，确认指标标签和数据字段接线存在。
- 运行前端完整测试和生产构建；运行 Rust `cargo test -j 1`，确认后端接口无回归。
