# 批量登录凭证错误快速跳过设计

## 目标

让企业账号密码输入过程可观察，并在 AWS 登录页明确提示凭证无效时立即关闭当前账号的独立 BrowserContext，记录失败后继续下一账号。

## 根因

`AccountBrowserSession._drive_login()` 当前在提交企业密码后立即返回，因此不再读取提交后的页面正文；同时 `INVALID_TEXT` 没有覆盖“无法验证您的登录凭证”一类 AWS 中文提示。结果是错误页保持打开，后台只剩 IdC token 轮询，直到用户取消或超时。

## 行为设计

- 密码字段使用 Playwright 顺序按键输入，默认每字符间隔 35ms，使输入长度和过程可见。
- 密码输入完成后停留 0.8 秒再点击登录。
- 企业登录提交后继续轮询页面，而不是立即把浏览器任务标记完成。
- 匹配中文或英文凭证无效提示时抛出 `BrowserFlowError(code="invalid_credentials")`。
- 匹配授权成功页面时正常返回；若页面文案未知，则 IdC token 轮询成功仍作为最终成功信号并取消页面监控任务。
- `LocalBatchRunner` 沿用现有单账号错误隔离：记录失败、关闭 BrowserContext、继续下一账号。

## 安全与日志

- 错误事件只包含稳定错误码，不包含账号、密码或页面正文。
- 不自动勾选网页上的“显示密码”，避免明文密码暴露。
- GUI 现有 `account_finished` 事件会记录 `invalid_credentials`，无需传递敏感页面内容。

## 验证

- 回归测试复现 AWS 中文“无法验证登录凭证”页面，确认抛出不可重试凭证错误。
- 浏览器合约测试确认密码产生逐字符输入事件。
- 现有企业成功、设备码、Microsoft、批次失败继续和全量测试必须通过。
