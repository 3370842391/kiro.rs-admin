# 账号 URL 自动识别设计

## 目标

“粘贴并识别账号”无需手动切换账号格式，即可识别：

```text
账号----密码----https://d-example.awsapps.com/start
```

## 设计

- 保留现有账号格式模板及其严格解析行为。
- GUI 账号导入显式启用自动识别；CLI 等其他调用方默认不改变。
- 当模板不含 `{start_url}` 时，每行先尝试
  `{account}----{password}----{start_url}`；匹配失败后再使用当前模板。
- 密码字段保持贪婪匹配，以最右侧 `----` 后的 URL 作为 Start URL，确保密码中的特殊字符和额外 `----` 原样保留。
- 自动识别出的 URL 继续使用现有 HTTPS、主机名及禁止内嵌凭据校验。
- 明确含 `{start_url}` 的模板不启用额外推断，避免覆盖用户指定的格式。

## 错误处理

- 自动格式结构匹配但 URL 非法时，报告 `invalid_start_url`。
- 两种格式都不匹配时，保持现有 `format_mismatch` 提示。
- 企业账号仍优先使用每行 URL；未识别到时继续使用顶部统一 Start URL。

## 测试

- 截图所示格式无需切换模板即可解析每行 URL。
- 密码包含特殊字符及额外 `----` 时完整保留。
- 未启用自动识别的调用方保持原有行为。
- 旧模板与显式 `{start_url}` 模板回归测试继续通过。
